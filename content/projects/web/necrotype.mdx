A generative design tool for metal logos etc. Check it out [here](https://necrotype.netlify.app/)

I've been developing an interest in automating creative processes: Many are directly algorithmic. I had a go at drawing a black/death metal style logo for my music project the other day and it was brutal (not in a good way). I guess after having been at a computer or thinking about math for years my brain automatically starts trying to find the most efficient way to get something done.

The process of drawing one of these types of logos is iterative and random: You start with text that is generally distorted, then you do a pass and extend randomly (within some parameter space, e.g. orientation and symmetry) branches/arms out of this text in some style. Repeat step two until satisfied.

I wrote this process out in quite a bit more detail, noting specific fields I thought would be important to have control over, and told claude to get to work. I may be giving myself too much credit but I think the breakdown of the process into concrete steps is what made the result such a success - the code faithfully captures the exact style that I was unable to draw myself, and makes some pretty weird and fun artifacts in the meantime (the "randomize all" button gets pretty weird).

## Pipeline

Four stages, each a pure function:

1. **Glyph acquisition**: opentype.js extracts cubic Bezier outlines for the chosen font, flattened to dense polylines per character. Bypasses the library's GSUB / Bidi shaping path (some otherwise-good fonts trip its substitution-format support).
2. **Per-glyph distortion**: affine perturbation (rotation / scale / shear) plus per-vertex coherent-noise displacement.
3. **Whole-logo transform**: global skew, vertical stretch, perspective tilt, sinusoidal wave, sinusoidal twist.
4. **Multi-pass directional extrusion**: sample roots on each contour, grow tendrils outward with curl + recursive branching. A spatial hash mediates overlap rejection so tendrils respect each other and letter boundaries; symmetry mirrors the just-grown tendril across the logo's vertical center.

The renderer bins segments by stroke width and chains adjacent segments into single sub-paths so round line caps appear only at real tendril endpoints. SVG export collapses everything into a single `<path>` with `fill-rule: nonzero`, dropping cleanly into Illustrator / Inkscape as one editable shape.

## Stack

- Vite 6, React 19, TypeScript (strict)
- Tailwind CSS v4 via `@tailwindcss/vite`
- opentype.js 1.3 for font parsing
- Single page, no router, no state library; same shell as this site

## Notes

- **"I'm feeling heavy"** randomizes within ranges chosen to produce visually-heavy results (high curl + symmetry + taper, mid stroke, low overlap, low branch). The image at the top of this post was generated this way: no slider tweaking, just press + re-roll the seed.
- **Time-budget anti-freeze**: at extreme parameter combos (high branch + low overlap + long tendrils) the pipeline can blow up in cost. A 700 ms full-render budget caps it; on overrun the pipeline bails with a partial result and the status bar flags it.
- **Variable fonts work**: `Cinzel[wght].ttf`, `Oswald[wght].ttf`, `Merriweather[opsz,wdth,wght].ttf`, `PlayfairDisplay[wght].ttf` load with URL-encoded brackets.
